    进程间通信

    Linux通过操作系统API如Linux的fork、exec、wait等来实现进程的创建、管理和通信。

    在 Linux 下，C/C++ 创建进程并执行新程序，常用的方式是先用 fork() 创建子进程，再用 exec 系列函数在子进程中执行新程序。

    典型代码示例

    #include <unistd.h>
#include <sys/types.h>
#include <sys/wait.h>
#include <iostream>
using namespace std;

int main() {
    pid_t pid = fork(); // 创建子进程
    if (pid == 0) {
        // 子进程
        execlp("ls", "ls", "-l", NULL); // 执行ls -l命令
        // 如果exec执行失败，才会执行下面这行
        perror("exec failed");
    } else if (pid > 0) {
        // 父进程
        wait(NULL); // 等待子进程结束
        cout << "子进程执行完毕" << endl;
    } else {
        // fork失败
        perror("fork failed");
    }
    return 0;
}

    说明

    • fork()：创建一个新进程，父进程返回子进程PID，子进程返回0。
    • execlp()：在子进程中执行新程序（如ls -l），成功后不会返回，失败才会继续执行后面的代码。
    • wait(NULL)：父进程等待子进程结束。

    常用的 exec 系列函数有： - execl、execlp、execv、execvp 等，区别在于参数传递方式和是否自动查找PATH。

    总结：
    Linux 下 C/C++ 创建进程并执行新程序，常用 fork() + exec()，父进程可用 wait() 等待子进程结束。

    进程间通信（IPC）常见方式有：

    1. 管道（pipe）
    2. 命名管道（FIFO）
    3. 信号（signal）
    4. 共享内存（shm）
    5. 消息队列（message queue）
    6. 套接字（socket）

    下面以匿名管道和共享内存为例，分别用C++代码简单演示：

    1. 匿名管道（pipe）

    #include <unistd.h>
#include <iostream>
using namespace std;

int main() {
    int fd[2];
    pipe(fd); // 创建管道
    pid_t pid = fork();
    if (pid == 0) {
        // 子进程：写数据
        close(fd[0]);
        const char* msg = "hello from child";
        write(fd[1], msg, 16);
        close(fd[1]);
    } else {
        // 父进程：读数据
        close(fd[1]);
        char buf[64] = {0};
        read(fd[0], buf, 64);
        cout << "父进程收到: " << buf << endl;
        close(fd[0]);
    }
    return 0;
}

    2. 共享内存（shm）

    #include <sys/ipc.h>
#include <sys/shm.h>
#include <unistd.h>
#include <cstring>
#include <iostream>
using namespace std;

int main() {
    int shmid = shmget(IPC_PRIVATE, 64, IPC_CREAT | 0666); // 创建共享内存
    pid_t pid = fork();
    if (pid == 0) {
        // 子进程：写数据
        char* shmaddr = (char*)shmat(shmid, nullptr, 0);
        strcpy(shmaddr, "hello from child");
        shmdt(shmaddr);
    } else {
        // 父进程：读数据
        sleep(1); // 等待子进程写入
        char* shmaddr = (char*)shmat(shmid, nullptr, 0);
        cout << "父进程收到: " << shmaddr << endl;
        shmdt(shmaddr);
        shmctl(shmid, IPC_RMID, nullptr); // 删除共享内存
    }
    return 0;
}

    总结：
    常见IPC方式有管道、共享内存、消息队列、信号、套接字等。实际开发中可根据需求选择合适的通信方式。

    信号（signal）是 Linux/Unix 系统中一种用于进程间通信和进程控制的机制，本质上是一种异步通知。操作系统或其他进程可以向目标进程发送信号，通知它发生了某种事件（如终止、暂停、继续、定时器到期等）。

    常见信号举例

    • SIGINT：中断信号（如按Ctrl+C）
    • SIGTERM：终止信号（请求程序正常终止）
    • SIGKILL：强制终止信号（无法被捕获或忽略）
    • SIGSTOP：暂停进程
    • SIGCONT：继续执行
    • SIGCHLD：子进程状态改变

    用法说明

    • 进程可以用kill命令或kill()函数向其他进程发送信号。
    • 进程可以通过注册信号处理函数（signal()或sigaction()）响应特定信号。

    简单C++代码示例

    #include <iostream>
#include <csignal>
#include <unistd.h>
using namespace std;

void handler(int signum) {
    cout << "收到信号: " << signum << endl;
    exit(0);
}

int main() {
    signal(SIGINT, handler); // 捕获Ctrl+C
    while (true) {
        cout << "运行中..." << endl;
        sleep(1);
    }
    return 0;
}

    总结：
    信号是一种轻量级的进程间通信方式，常用于通知、控制进程状态、异常处理等场景。

    在 Linux 下，可以通过以下几种方式查看各种进程的状态：

    1. 使用 ps 命令

    ps aux
    或 
    ps -ef
    - 这会列出所有进程及其状态（STAT列）。

    常见状态码（STAT列）： - R：运行（Running） - S：可中断睡眠（Sleeping） - D：不可中断睡眠（Disk sleep） - T：停止（Stopped） - Z：僵尸（Zombie）

    2. 使用 top 或 htop

    top
    - 实时显示进程状态，S、R、Z等含义同上。

    3. 查看 proc 目录

    每个进程在 /proc/[pid]/status 文件中有详细状态信息：

    cat /proc/1/status
    - 其中 State: 一行显示该进程的状态。

    总结：
    - ps、top、htop 等命令都可以查看进程状态。 - 进程状态常见有：R（运行）、S（睡眠）、D（不可中断）、T（停止）、Z（僵尸）等。

    引入进程的原因，是为了实现多道程序设计，让多个程序能在同一台计算机上“同时”运行，提高资源利用率和系统响应速度。进程是操作系统进行资源分配和调度的基本单位。

    什么是进程

    进程（Process）是一个正在运行的程序的实例，是操作系统资源分配和调度的基本单位。每个进程拥有独立的地址空间、数据、代码和系统资源。

    进程由什么组成

    1. 程序代码
      进程要执行的指令（可执行文件的代码段）。

    2. 数据段
      包括全局变量、静态变量、堆、栈等。

    3. 进程控制块（PCB）
      操作系统用来管理和描述进程的数据结构，包含：

    4. 进程状态（如运行、就绪、阻塞等）
    5. 程序计数器（PC）
    6. 寄存器内容
    7. 进程号（PID）
    8. 所占用的资源（如打开的文件、内存等）

    9. 调度信息和优先级

    10. 系统资源
      如打开的文件、信号量、消息队列、共享内存等。

    总结：
    - 进程是操作系统资源分配和调度的基本单位。 - 进程由程序代码、数据、PCB和系统资源等组成。

    查看进程控制块PCB

    在 Linux 下进程控制块的大部分信息都可以通过 /proc/[pid]/ 目录下的文件查看，这实际上就是操作系统暴露给用户的PCB内容。

    常用命令如下：

    1. 查看进程状态和关键信息

    cat /proc/<pid>/status
    - 包含进程名、状态、PID、PPID、UID、内存、线程数等。

    2. 查看进程详细信息

    cat /proc/<pid>/stat
    - 包含进程的所有核心字段（如状态、优先级、CPU时间、内存等）。

    进程控制块（PCB）的主要信息。下面对常见字段做简要解释：

    • Name：进程名
    • Umask：文件创建掩码
    • State：进程状态（如 I=idle，R=running，S=sleeping，Z=zombie 等）
    • Tgid：线程组ID（一般等于主线程的PID）
    • Ngid：NUMA组ID（一般为0）
    • Pid：进程ID
    • PPid：父进程ID
    • TracerPid：跟踪该进程的进程ID（如被调试时）
    • Uid/Gid：用户ID/组ID（4列分别为实际、有效、保存的、文件系统的ID）
    • FDSize：文件描述符表大小
    • Groups：所属的组
    • NStgid/NSpid/NSpgid/NSsid：命名空间相关的进程/组/会话ID
    • Threads：该进程的线程数
    • SigQ：信号队列（已占用/最大）
    • SigPnd/ShdPnd：进程/线程的待处理信号
    • SigBlk/SigIgn/SigCgt：被阻塞/忽略/捕获的信号掩码
    • CapInh/CapPrm/CapEff/CapBnd/CapAmb：进程的各种能力（capabilities）掩码
    • NoNewPrivs：是否禁止新特权
    • Seccomp：安全计算模式
    • Speculation_Store_Bypass：CPU投机执行安全状态
    • Cpus_allowed/Cpus_allowed_list：允许在哪些CPU上运行
    • Mems_allowed/Mems_allowed_list：允许在哪些内存节点上运行
    • voluntary_ctxt_switches：自愿上下文切换次数
    • nonvoluntary_ctxt_switches：非自愿上下文切换次数

    这些字段共同描述了进程的基本信息、资源、权限、状态等，是操作系统管理进程的重要数据。

    3. 查看进程打开的文件

    ls /proc/<pid>/fd
    - 显示该进程打开的所有文件描述符。

    4. 查看进程的内存映射

    cat /proc/<pid>/maps

    总结：
    - /proc/[pid]/status 和 /proc/[pid]/stat 就是PCB信息的主要来源。 - 没有单独的“pcb”命令，都是通过 proc 伪文件系统来查看。

    在linux子进程是父进程fork的，那这个巨大的进程网络都是由一个最开始的进程产生吗，我想知道操作系统启动的时候这些进程是怎么调度的

    你的理解是对的！在Linux系统中，所有用户进程最终都是由一个“最初的进程”祖先衍生出来的。这个“最初的进程”就是init进程（在现代系统中通常是systemd，PID为1）。

    Linux启动和进程树的形成

    1. 内核启动
      计算机加电后，加载并启动Linux内核。

    2. 内核创建第一个进程（PID 1）
      内核会手动创建第一个用户空间进程，传统上叫init，现代Linux多为systemd，它的PID永远是1。

    3. init（systemd）负责启动其他进程

    4. init/systemd会根据配置文件启动各种系统服务、守护进程、登录终端等。

    5. 这些服务进程又可以fork出自己的子进程，形成一棵进程树。

    6. 用户登录后shell等进程被启动
      用户登录后，shell（如bash）被启动，用户在shell中运行的每个程序，都是shell的子进程。

    进程调度

    • 调度由内核负责
      内核中的调度器（scheduler）负责决定哪个进程/线程什么时候运行，分配CPU时间片。
    • 调度算法
      Linux采用CFS（完全公平调度器）等算法，保证进程公平、高效地使用CPU。
    • 进程状态切换
      内核会根据进程状态（就绪、运行、阻塞等）动态切换进程，保证系统响应和资源利用率。

    进程树示意

    systemd/init (PID 1)
 ├─ 服务A
 ├─ 服务B
 ├─ login
 │   └─ bash
 │       └─ 用户程序
 └─ ...

    总结：
    - Linux所有进程最终都可追溯到PID 1（init/systemd）。 - 内核负责进程调度，保证多进程并发运行。 - 进程树结构清晰，父子关系明确，系统启动时由init/systemd逐步派生出整个进程网络。
