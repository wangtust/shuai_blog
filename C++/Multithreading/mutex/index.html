    线程互斥与同步

    mutex 的全称是 mutual exclusion，中文意思是“互斥”。

    • mutual：相互的
    • exclusion：排斥、排除

    mutex（互斥锁）就是用来保证同一时刻只有一个线程能访问某个资源，从而避免多线程同时操作导致的数据混乱和竞态条件。

    在C++中，线程的互斥与同步主要通过互斥锁（mutex）和条件变量（condition_variable）来实现。

    1. 互斥（mutex）

    作用：保证同一时刻只有一个线程能访问共享资源，防止数据竞争。

    基本用法

    #include <iostream>
#include <thread>
#include <mutex>
using namespace std;

mutex mtx; // 定义一个互斥锁

void print(int id) {
    mtx.lock(); // 加锁
    cout << "Thread " << id << endl;
    mtx.unlock(); // 解锁
}

int main() {
    thread t1(print, 1);
    thread t2(print, 2);
    t1.join();
    t2.join();
    return 0;
}
    在这段代码中：

    thread t1(print, 1);
thread t2(print, 2);
    • print 是线程要执行的函数名。
    • 1 和 2 是传递给 print 函数的参数。

    也就是说，t1 线程会执行 print(1)，t2 线程会执行 print(2)。
    这样每个线程都能拿到不同的参数，输出不同的内容。

    推荐写法： 

    void print(int id) {
    lock_guard<mutex> lock(mtx); // 自动加锁和解锁
    cout << "Thread " << id << endl;
}

    为什么去掉mtx.lock()和mtx.unlock()也没事

    现在的代码里，print函数只是简单地输出一句话：

    void print(int id) {
    cout << "Thread " << id << endl;
}

    为什么去掉mtx也没事？

    因为： - cout << ... 这行代码很短，两个线程几乎不会同时写入同一行，出现输出混乱的概率很低。 - 即使有时输出顺序错乱（比如两行内容交叉在一起），对程序功能没有影响，只是显示上不整齐。

    但实际上： - cout 不是线程安全的，如果多个线程同时输出，可能会出现内容交叉、乱码等问题。 - 如果临界区（共享资源操作）变复杂，比如多个线程同时修改同一个变量，没有加锁就会出错。

    总结：
    本例中去掉mtx没事，是因为操作简单且影响不大。但在实际多线程开发中，只要有共享资源操作，都应该加锁保护，否则容易出现难以发现的竞态条件和数据错误。

    找个程序观察加锁和不加锁的区别

    在多线程环境下，绝大多数竞态条件都是“概率性”出错，不加锁时“必定出错”的例子很难保证每次都复现。但可以通过增加线程数量、循环次数、或者让线程sleep制造竞争，极大提高出错概率。

    让写操作变慢

    void setA() {
    for (int i = 0; i < 7; ++i) {
        d.buf[i] = 'A';
        std::this_thread::sleep_for(std::chrono::nanoseconds(1));
    }
    d.buf[7] = '\0';
}

void setB() {
    for (int i = 0; i < 7; ++i) {
        d.buf[i] = 'B';
        std::this_thread::sleep_for(std::chrono::nanoseconds(1));
    }
    d.buf[7] = '\0';
}
    这样每次写一个字符，增加线程切换的机会，更容易出现混合字符串（如AABBBBA等）。

    这个程序很好地演示了多线程不加锁导致数据竞争和结果不可预测的现象。下面是详细解释：

    程序结构

    • 有一个全局结构体 Data d，包含一个字符数组 buf[8]。
    • 有两个线程函数 setA() 和 setB()，分别把 d.buf 的前7个字符写成 'A' 或 'B'，每写一个字符都 sleep 一下，最后加上字符串结束符 '\0'。
    • 在 main() 中，分别启动两个线程 t1(setA) 和 t2(setB)，并等待它们结束，然后输出 d.buf。

    运行原理

    • setA 和 setB 两个线程同时操作同一块内存（d.buf），但没有加锁保护。
    • 每个线程写一个字符后都会短暂休眠，人为制造了线程切换的机会，这样两个线程的写操作就很容易交叉进行。
    • 由于没有互斥保护，d.buf 可能被两个线程交替写入，导致最终内容既不是全A，也不是全B，而是混合字符串（如AABBBBA、BBBBBAA等），甚至可能出现乱码。

    会观察到什么？

    • 有时输出是 AAAAAAA 或 BBBBBBB（某个线程“赢了”）。
    • 很多时候输出是混合的，比如 AABBBBB、BBBAAAA、ABABABA 等。
    • 这就是数据竞争和竞态条件的直观体现。

    总结

    • 这个例子说明：多线程同时写同一块内存，不加锁就会导致数据混乱，结果不可预测。
    • 在实际开发中，遇到类似共享资源的操作，一定要加锁保护，否则会出现难以发现和复现的bug。

    如需加锁保护，只需在setA和setB里加上mutex即可解决这个问题。

    总结

    • 多线程写同一块内存，不加锁理论上一定会出错，但实际“出错”现象出现的概率和环境、写入速度、线程调度有关。
    • 用循环多次和逐字节写入+延迟，可以极大提高出错概率，更容易观察到问题。

    2. 同步（condition_variable）

    作用：让一个线程等待某个条件成立，由另一个线程通知它继续执行，常用于生产者-消费者模型。

    基本用法

    #include <iostream>
#include <thread>
#include <mutex>
using namespace std;

mutex mtx;        // 全局互斥锁，保护ready变量
bool ready = false; // 线程同步标志

// 工作线程函数，等待ready变为true再继续
void worker() {
    while (true) {
        mtx.lock(); // 加锁，安全访问ready
        if (ready) {
            mtx.unlock(); // 解锁
            break;        // ready为true，退出循环
        }
        mtx.unlock(); // 解锁
        this_thread::sleep_for(chrono::milliseconds(10)); // 休眠一会儿，避免CPU空转
    }
    cout << "Worker thread running!" << endl;
}

int main() {
    thread t(worker); // 创建并启动工作线程
    this_thread::sleep_for(chrono::seconds(1)); // 主线程休眠1秒，模拟准备工作
    mtx.lock();      // 加锁
    ready = true;    // 设置ready为true，通知工作线程
    mtx.unlock();    // 解锁
    t.join();        // 等待工作线程结束
    return 0;
}

    总结： - 互斥锁（mutex）用于保护共享资源，防止竞态条件。 - 条件变量（condition_variable）用于线程间的同步与通信。

    condition_variable（条件变量）一般用于线程间同步，让一个或多个线程等待某个条件成立，由另一个线程通知它们继续执行。常见于生产者-消费者模型等场景。

    标准用法

    #include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
using namespace std;

mutex mtx;
condition_variable cv;
bool ready = false;

void worker() {
    unique_lock<mutex> lock(mtx);           // 必须用unique_lock
    cv.wait(lock, []{ return ready; });     // 等待ready为true
    cout << "Worker thread running!" << endl;
}

int main() {
    thread t(worker);
    this_thread::sleep_for(chrono::seconds(1)); // 主线程模拟准备
    {
        lock_guard<mutex> lock(mtx);
        ready = true;                         // 设置条件
    }
    cv.notify_one();                          // 通知等待的线程
    t.join();
    return 0;
}

    用法说明

    1. 等待方（如worker线程）：
    2. 用unique_lock<mutex>加锁。

    3. 调用cv.wait(lock, 条件)，如果条件不满足就阻塞等待，直到被通知且条件满足。

    4. 通知方（如主线程）：

    5. 修改条件变量前加锁，设置条件。
    6. 调用cv.notify_one()或cv.notify_all()通知等待线程。

    总结： - condition_variable配合mutex和条件标志使用，实现线程间高效同步与通信。 - 推荐用cv.wait(lock, 条件)这种写法，避免虚假唤醒。
