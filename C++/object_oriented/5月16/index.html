    5月16日基础知识

    纯虚函数有什么作用？如何实现？

    纯虚函数的作用是实现接口（抽象类），用于规定子类必须实现某些函数，从而实现多态。

    作用

    • 让基类成为抽象类，不能实例化，只能作为接口使用。
    • 强制所有派生类实现该函数，实现多态。

    如何实现

    在类中声明纯虚函数，语法是在函数声明后加= 0，例如：

    class Base {
public:
    virtual void func() = 0; // 纯虚函数
};

    只要类中有纯虚函数，这个类就是抽象类，不能直接创建对象。派生类必须重写所有纯虚函数，才能实例化对象。

    纯虚函数最常用的方法是用来实现接口类，让所有子类都必须实现某些功能。例如，定义一个动物基类，所有动物都必须实现makeSound()方法：

    class Animal {
public:
    virtual void makeSound() = 0; // 纯虚函数
};

class Dog : public Animal {
public:
    void makeSound() override {
        cout << "Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() override {
        cout << "Meow!" << endl;
    }
};

    这样，Animal类不能实例化，所有派生类都必须实现makeSound()，实现了多态。

    在C++中，派生类是通过继承基类来定义的。语法如下：

    class 派生类名 : public 基类名 {
    // 派生类自己的成员和方法
};

    在上面例子中，Dog和Cat都是从Animal基类公有继承（public Animal）而来的：

    class Dog : public Animal {
public:
    void makeSound() override {
        cout << "Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() override {
        cout << "Meow!" << endl;
    }
};

    说明： - : public Animal 表示公有继承，基类的public和protected成员在派生类中保持原有访问权限。 - 派生类必须实现基类中的所有纯虚函数，否则派生类也会成为抽象类，不能实例化。

    总结语法： 

    class 派生类名 : public 基类名 {
    // 派生类成员
};

    更通俗理解纯虚函数作用

    纯虚函数就是规定“子类必须实现某个功能”，它让基类变成“模板”或“接口”，只定义规范，不写具体实现。这样，所有继承它的子类都必须自己写出具体实现。

    举个生活中的例子

    假如你有一个“动物”基类，里面有一个“叫声”功能，但每种动物叫声都不一样，所以你只规定“必须有叫声”，但不写怎么叫：

    class Animal {
public:
    virtual void makeSound() = 0; // 纯虚函数，只规定接口，不写内容
};

    这就像你规定：所有动物都要会叫，但怎么叫你不管。

    然后，狗和猫继承动物，都必须自己写出怎么叫：

    class Dog : public Animal {
public:
    void makeSound() override { cout << "Woof!" << endl; }
};

class Cat : public Animal {
public:
    void makeSound() override { cout << "Meow!" << endl; }
};

    这样做的好处

    • 统一规范：所有动物都必须有makeSound()，不会忘记实现。
    • 多态：你可以用Animal* 指针指向任何动物对象，调用makeSound()时会自动调用对应的实现。

    代码示例

    Animal* a1 = new Dog();
Animal* a2 = new Cat();
a1->makeSound(); // 输出 Woof!
a2->makeSound(); // 输出 Meow!

    你不用关心a1、a2到底是什么动物，只要它们是Animal的子类，就一定有makeSound()功能。

    总结：
    纯虚函数就是用来“规定接口”，让所有子类都必须实现某些功能，实现统一规范和多态。

    虚函数和纯虚函数区别，虚函数又是做什么用的

    虚函数

    • 定义：在基类中用virtual关键字声明的成员函数。
    • 作用：实现多态。允许通过基类指针或引用调用派生类中重写的函数。
    • 实现：基类可以给出默认实现，派生类可以选择重写，也可以不重写。
    • 语法： 
      class Base {
public:
    virtual void func() { cout << "Base" << endl; }
};

    纯虚函数

    • 定义：在基类中用virtual声明，并在末尾加= 0，没有函数体。
    • 作用：规定接口，强制所有派生类必须实现该函数，基类变成抽象类，不能实例化。
    • 实现：基类不能给出实现，派生类必须重写。
    • 语法： 
      class Base {
public:
    virtual void func() = 0; // 纯虚函数
};

    区别总结

    虚函数 纯虚函数
    是否有实现 可以有实现 不能有实现（=0）
    是否必须重写 可以不重写 派生类必须重写
    基类能否实例化 可以 不能（抽象类）
    作用 支持多态 规定接口+支持多态

    虚函数的用途举例

    class Animal {
public:
    virtual void makeSound() { cout << "Some sound" << endl; }
};

class Dog : public Animal {
public:
    void makeSound() override { cout << "Woof!" << endl; }
};

Animal* a = new Dog();
a->makeSound(); // 输出 Woof!

    如果makeSound不是虚函数，则a->makeSound()会调用基类的实现。

    总结： - 虚函数：用于多态，基类可以有默认实现，派生类可选重写。 - 纯虚函数：用于接口规范，基类不能实例化，派生类必须实现。

    为什么叫“虚函数”？

    • “虚”字的本意是“不是固定的”，也就是不确定最终会调用哪个实现。
    • 虚函数在基类里声明时，只是告诉编译器：“这个函数以后可能会被子类重写（覆盖）。”
    • 当你用基类指针或引用调用虚函数时，实际运行时会根据对象的真实类型，动态决定调用哪个版本（这就是多态）。
    • 所以叫“虚函数”，意思是“函数的具体实现是‘虚’的、不确定的，要等到运行时才能确定”。

    为什么叫“纯虚函数”？

    • “纯”表示“纯粹的、完全的”，意思是这个函数在基类里完全没有实现，只规定了接口。
    • 纯虚函数就是“纯粹的虚函数”，它连默认实现都没有，只能由子类来实现。
    • 这样，基类就成了“纯粹的接口”，不能实例化，只能被继承。

    总结

    • 虚函数：有实现，也可以被子类重写，调用时“虚”指向谁要等运行时决定。
    • 纯虚函数：没有实现，只规定接口，必须由子类实现，基类成了“纯粹的虚类”（抽象类）。

    你可以理解为： - “虚”=“不确定、可变、运行时决定” - “纯虚”=“完全没有实现，只能由子类实现”

    这样命名是为了突出它们的“动态性”和“接口性”。

    虚函数表是针对类的还是针对对象的？同一个类的两个对象的虚函数表是怎么维护的？

    虚函数表（vtable）是针对类的，不是针对每个对象单独维护的。

    详细解释

    • 每个含有虚函数的类，编译器会为它生成一张虚函数表（vtable），表里存放着虚函数的地址。
    • 同一个类的所有对象共用同一张虚函数表，不会为每个对象单独生成。
    • 每个对象内部会有一个指向虚函数表的指针（通常叫vptr），这个指针在对象创建时由编译器自动设置，指向所属类的虚函数表。

    举例说明

    class Base {
public:
    virtual void func() { cout << "Base" << endl; }
};

class Derived : public Base {
public:
    void func() override { cout << "Derived" << endl; }
};

Base b1, b2;
    • Base类有一张虚函数表，b1和b2对象内部都有一个vptr，指向Base的虚函数表。
    • 如果有Derived d1, d2;，它们的vptr会指向Derived的虚函数表。

    总结

    • 虚函数表是类级别的，一个类一张表。
    • 同一个类的所有对象共用同一张虚函数表，只是每个对象内部有一个指针指向这张表。
