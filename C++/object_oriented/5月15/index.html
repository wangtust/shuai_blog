    面向对象的三大特征

    1. 封装（Encapsulation）：
    2. 定义：
      • 封装是将数据（属性）和操作数据的函数（方法）绑定在一起，隐藏对象的内部实现细节，只对外暴露必要的接口。
    3. 作用：
      • 提高代码的安全性，防止外部直接访问和修改对象的内部数据。
      • 提供清晰的接口，方便使用和维护。
    4. 示例： 
      #include <iostream>
using namespace std;

class Person {
private:
    string name; // 私有属性
public:
    void setName(string n) { // 公有方法
        name = n;
    }
    string getName() {
        return name;
    }
};

int main() {
    Person p;
    p.setName("Alice"); // 通过方法设置属性
    cout << p.getName() << endl; // 通过方法获取属性
    return 0;
}
    1. 继承（Inheritance）：
    2. 定义：
      • 继承是一个类（子类）从另一个类（父类）获取属性和方法的机制，子类可以复用父类的代码，也可以扩展或重写父类的功能。
    3. 作用：
      • 提高代码的复用性，减少重复代码。
      • 实现类之间的层次结构，便于扩展和维护。
    4. 示例： 
      #include <iostream>
using namespace std;

class Animal {
public:
    void eat() {
        cout << "I can eat!" << endl;
    }
};

class Dog : public Animal { // Dog 继承自 Animal
public:
    void bark() {
        cout << "I can bark!" << endl;
    }
};

int main() {
    Dog d;
    d.eat(); // 调用父类方法
    d.bark(); // 调用子类方法
    return 0;
}
    1. 多态（Polymorphism）：
    2. 定义：
      • 多态是指同一个接口可以表现出不同的行为，分为编译时多态（函数重载、运算符重载）和运行时多态（虚函数）。
    3. 作用：
      • 提高代码的灵活性和可扩展性。
      • 通过统一接口实现不同的功能。
    4. 示例： 
      #include <iostream>
using namespace std;

class Animal {
public:
    virtual void speak() { // 虚函数
        cout << "Animal speaks" << endl;
    }
};

class Dog : public Animal {
public:
    void speak() override { // 重写父类方法
        cout << "Dog barks" << endl;
    }
};

int main() {
    Animal* a = new Dog(); // 父类指针指向子类对象
    a->speak(); // 调用子类的重写方法
    delete a;
    return 0;
}

    总结

    特性 定义 作用 示例
    封装 将数据和方法绑定在一起，隐藏实现细节，只暴露必要接口 提高安全性，便于使用和维护 私有属性 + 公有方法
    继承 子类继承父类的属性和方法，复用代码 提高代码复用性，便于扩展和维护 子类继承父类，扩展或重写功能
    多态 同一接口表现不同行为，分为编译时多态和运行时多态 提高灵活性和可扩展性 函数重载、虚函数

    这三大特性是面向对象编程的核心，帮助开发者编写更安全、可维护、可扩展的代码。

    C++ 中类成员的访问权限

    C++ 类成员的访问权限通过 访问控制符 来定义，主要有以下三种：

    1. public（公有）

    • 定义：
    • 公有成员可以被类的外部直接访问。
    • 特点：
    • 没有访问限制，任何地方都可以访问。
    • 通常用于定义类的接口（如公有方法）。
    • 示例： 
      #include <iostream>
using namespace std;

class Person {
public:
    string name; // 公有成员
    void display() {
        cout << "Name: " << name << endl;
    }
};

int main() {
    Person p;
    p.name = "Alice"; // 直接访问公有成员
    p.display();      // 调用公有方法
    return 0;
}

    2. private（私有）

    • 定义：
    • 私有成员只能被类的内部访问，不能被类的外部直接访问。
    • 特点：
    • 默认访问权限（如果未指定访问控制符，成员默认为 private）。
    • 通常用于保护类的内部实现细节。
    • 示例： 
      #include <iostream>
using namespace std;

class Person {
private:
    string name; // 私有成员
public:
    void setName(string n) {
        name = n; // 通过公有方法访问私有成员
    }
    void display() {
        cout << "Name: " << name << endl;
    }
};

int main() {
    Person p;
    // p.name = "Alice"; // 错误：无法直接访问私有成员
    p.setName("Alice"); // 通过公有方法设置私有成员
    p.display();
    return 0;
}

    3. protected（受保护）

    • 定义：
    • 受保护成员可以被类的内部和子类访问，但不能被类的外部直接访问。
    • 特点：
    • 主要用于继承，允许子类访问父类的受保护成员。
    • 示例： 
      #include <iostream>
using namespace std;

class Animal {
protected:
    string species; // 受保护成员
public:
    void setSpecies(string s) {
        species = s;
    }
};

class Dog : public Animal {
public:
    void display() {
        cout << "Species: " << species << endl; // 子类可以访问受保护成员
    }
};

int main() {
    Dog d;
    d.setSpecies("Canine");
    d.display();
    return 0;
}

    4. 访问权限总结

    访问控制符 类内部 子类 类外部
    public 可以访问 可以访问 可以访问
    protected 可以访问 可以访问 不可访问
    private 可以访问 不可访问 不可访问

    5. 注意事项

    1. 默认访问权限：
    2. 类的成员默认是 private。

    3. 结构体（struct）的成员默认是 public。

    4. 继承中的访问权限：

    5. 在继承时，基类的访问权限可能会受到继承方式（public、protected、private）的影响。

    6. 封装性：

    7. 使用 private 和 protected 可以隐藏类的实现细节，增强封装性。

    6. 示例：综合使用访问权限

    #include <iostream>
using namespace std;

class Base {
private:
    int privateVar; // 私有成员
protected:
    int protectedVar; // 受保护成员
public:
    int publicVar; // 公有成员

    Base() : privateVar(1), protectedVar(2), publicVar(3) {}
};

class Derived : public Base {
public:
    void display() {
        // cout << privateVar; // 错误：无法访问私有成员
        cout << "Protected: " << protectedVar << endl; // 可以访问受保护成员
        cout << "Public: " << publicVar << endl;       // 可以访问公有成员
    }
};

int main() {
    Derived d;
    d.display();
    // cout << d.protectedVar; // 错误：无法访问受保护成员
    cout << "Public: " << d.publicVar << endl; // 可以访问公有成员
    return 0;
}

    输出： 

    Protected: 2
Public: 3
Public: 3

    总结

    • public：对所有地方可见，通常用于类的接口。
    • private：仅对类的内部可见，保护类的实现细节。
    • protected：对子类可见，通常用于继承场景。

    合理使用访问控制符可以提高代码的安全性、可维护性和封装性。

    C++ 中多态的实现方式

    C++ 中多态分为两种类型：编译时多态 和 运行时多态。以下是它们的实现方式：

    1. 编译时多态（静态多态）

    编译时多态是在编译阶段确定调用哪个函数，主要通过函数重载和运算符重载实现。

    1.1 函数重载

    • 定义：
    • 同一个作用域内定义多个同名函数，但参数列表不同（参数的类型、数量或顺序不同）。
    • 特点：
    • 编译器根据函数调用时的参数类型和数量选择合适的函数。
    • 示例： 
      #include <iostream>
using namespace std;

class Calculator {
public:
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }
};

int main() {
    Calculator calc;
    cout << calc.add(3, 5) << endl;       // 调用 int 版本
    cout << calc.add(2.5, 3.5) << endl;   // 调用 double 版本
    return 0;
}

    1.2 运算符重载

    • 定义：
    • 重定义 C++ 中的运算符，使其适用于用户自定义类型。
    • 特点：
    • 通过重载运算符，可以为类对象定义特定的运算行为。
    • 示例： 
      #include <iostream>
using namespace std;

class Complex {
public:
    double real, imag;

    Complex(double r, double i) : real(r), imag(i) {}

    // 重载加法运算符
    Complex operator+(const Complex& c) {
        return Complex(real + c.real, imag + c.imag);
    }

    void display() {
        cout << real << " + " << imag << "i" << endl;
    }
};

int main() {
    Complex c1(1.2, 3.4), c2(2.3, 4.5);
    Complex c3 = c1 + c2; // 使用重载的加法运算符
    c3.display();
    return 0;
}

    2. 运行时多态（动态多态）

    运行时多态是在程序运行时根据对象的实际类型决定调用哪个函数，主要通过虚函数实现。

    2.1 虚函数

    • 定义：
    • 在基类中使用 virtual 关键字声明的函数，允许子类重写。
    • 通过基类指针或引用调用虚函数时，会根据对象的实际类型调用对应的函数。
    • 特点：
    • 依赖虚函数表（vtable）实现。
    • 必须通过指针或引用调用，直接调用对象的虚函数不会表现出多态。
    • 示例： 
      #include <iostream>
using namespace std;

class Animal {
public:
    virtual void speak() { // 虚函数
        cout << "Animal speaks" << endl;
    }
};

class Dog : public Animal {
public:
    void speak() override { // 重写虚函数
        cout << "Dog barks" << endl;
    }
};

class Cat : public Animal {
public:
    void speak() override { // 重写虚函数
        cout << "Cat meows" << endl;
    }
};

int main() {
    Animal* a1 = new Dog();
    Animal* a2 = new Cat();

    a1->speak(); // 调用 Dog 的 speak()
    a2->speak(); // 调用 Cat 的 speak()

    delete a1;
    delete a2;
    return 0;
}

    输出： 

    Dog barks
Cat meows

    3. 编译时多态与运行时多态的对比

    特性 编译时多态 运行时多态
    实现方式 函数重载、运算符重载 虚函数
    绑定时间 编译时绑定 运行时绑定
    性能 无运行时开销 有一定的运行时开销（虚函数表查找）
    灵活性 灵活性较低 灵活性较高
    适用场景 参数类型已知，行为固定的场景 需要根据对象的实际类型决定行为的场景

    4. 总结

    C++ 中多态的实现方式主要有两种： 1. 编译时多态： - 通过函数重载和运算符重载实现。 - 在编译阶段确定调用哪个函数。 - 性能较高，但灵活性较低。

    1. 运行时多态：
    2. 通过虚函数实现。
    3. 在运行阶段根据对象的实际类型决定调用哪个函数。
    4. 性能稍低，但灵活性较高。

    两种多态各有适用场景，编译时多态适合静态行为，运行时多态适合动态行为。

    动态绑定的实现原理

    动态绑定（Dynamic Binding）是 C++ 中实现运行时多态的核心机制。它允许程序在运行时根据对象的实际类型调用对应的函数，而不是在编译时决定调用哪个函数。动态绑定主要通过 虚函数表（vtable） 和 虚函数指针（vptr） 实现。

    动态绑定的实现步骤

    1. 虚函数表（vtable）

    • 定义：
    • 虚函数表是一个由编译器生成的表格，存储类中所有虚函数的地址。

    • 每个类都有一个虚函数表，表中记录了该类的虚函数实现。

    • 特点：

    • 如果一个类没有虚函数，则不会生成虚函数表。
    • 如果一个类有虚函数，编译器会为该类生成一个虚函数表。

    2. 虚函数指针（vptr）

    • 定义：
    • 虚函数指针是一个隐藏的指针，存储在每个对象中，指向该对象所属类的虚函数表。

    • 每个对象都有一个虚函数指针，用于动态绑定虚函数。

    • 特点：

    • 当对象被创建时，虚函数指针会被初始化，指向该对象所属类的虚函数表。
    • 如果对象的类型发生变化（如通过继承），虚函数指针会指向新的虚函数表。

    3. 动态绑定的过程

    • 当通过基类指针或引用调用虚函数时：
    • 程序通过对象的虚函数指针（vptr）找到对应的虚函数表（vtable）。
    • 在虚函数表中查找虚函数的地址。
    • 调用虚函数表中存储的函数地址，执行实际的函数实现。

    动态绑定的示例

    代码示例

    #include <iostream>
using namespace std;

class Base {
public:
    virtual void speak() { // 虚函数
        cout << "Base speaks" << endl;
    }
    virtual ~Base() {} // 虚析构函数，确保正确释放资源
};

class Derived : public Base {
public:
    void speak() override { // 重写虚函数
        cout << "Derived speaks" << endl;
    }
};

int main() {
    Base* obj = new Derived(); // 基类指针指向子类对象
    obj->speak(); // 动态绑定，调用 Derived 的 speak()
    delete obj;
    return 0;
}

    输出

    Derived speaks

    动态绑定的实现细节

    1. 虚函数表的结构

    • 对于 Base 类，虚函数表可能如下： 

      vtable for Base:
+------------------+
| &Base::speak     | -> 地址指向 Base::speak 的实现
+------------------+

    • 对于 Derived 类，虚函数表可能如下： 

      vtable for Derived:
+------------------+
| &Derived::speak  | -> 地址指向 Derived::speak 的实现
+------------------+

    2. 虚函数指针的工作

    • 当 Base* obj = new Derived(); 执行时：
    • obj 的虚函数指针（vptr）会指向 Derived 类的虚函数表。
    • 调用 obj->speak() 时，程序通过 vptr 找到 Derived 的虚函数表，并调用 Derived::speak()。

    动态绑定的优缺点

    优点

    1. 灵活性：
    2. 动态绑定允许程序在运行时根据对象的实际类型调用函数，实现运行时多态。
    3. 代码复用：
    4. 通过基类指针或引用操作子类对象，减少代码重复。
    5. 扩展性：
    6. 新增子类时无需修改基类代码，符合开闭原则。

    缺点

    1. 性能开销：
    2. 动态绑定需要通过虚函数指针查找虚函数表，增加了运行时开销。
    3. 复杂性：
    4. 动态绑定依赖虚函数表和虚函数指针，增加了编译器的实现复杂性。
    5. 调试难度：
    6. 动态绑定的行为在运行时决定，可能增加调试的难度。

    动态绑定的注意事项

    1. 虚函数必须通过指针或引用调用：

    2. 如果直接通过对象调用虚函数，编译器会进行静态绑定。 

      Derived d;
d.speak(); // 静态绑定，直接调用 Derived::speak()

    3. 虚函数表的开销：

    4. 每个类有一个虚函数表，每个对象有一个虚函数指针，可能增加内存开销。

    5. 虚析构函数：

    6. 如果基类有虚函数，建议将析构函数声明为虚函数，以确保正确释放子类资源。 
      virtual ~Base() {}

    总结

    • 动态绑定的核心：
    • 通过虚函数表（vtable）和虚函数指针（vptr）实现。

    • 在运行时根据对象的实际类型调用对应的函数。

    • 实现过程：

    • 对象的虚函数指针指向所属类的虚函数表。
    • 调用虚函数时，通过虚函数指针找到虚函数表。

    • 在虚函数表中查找函数地址并调用。

    • 适用场景：

    • 动态绑定适用于需要根据对象的实际类型执行不同行为的场景，是实现运行时多态的关键机制。
