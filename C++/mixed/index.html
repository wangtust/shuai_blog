    C++知识点杂谈

    c++知识点杂谈

    保留两位小数

    要在 C++ 中输出指定位数的小数，可以按照以下步骤操作：

    步骤 1：包含头文件

    需要包含 <iomanip> 头文件： 

    #include <iomanip>

    步骤 2：使用 fixed 和 setprecision

    • fixed：强制以固定的小数点格式输出，而不是科学计数法。
    • setprecision(n)：设置小数点后保留 n 位。

    示例： 

    #include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double value = 3.14159;
    cout << fixed << setprecision(2) << value << endl; // 输出 3.14
    return 0;
}

    步骤 3：全局设置（可选）

    如果需要全局设置小数点格式，可以在程序开头设置一次： 

    cout << fixed << setprecision(2);
    之后所有浮点数输出都会保留两位小数。

    注意事项

    1. fixed 必须与 setprecision 一起使用：
    2. 如果只使用 setprecision，会设置浮点数的总有效数字，而不是小数点后的位数。
    3. 确保包含 <iomanip>：
    4. 如果未包含 <iomanip>，setprecision 和 fixed 无法使用。

    ++x 和 x++

    1. ++x（前置自增）

    • 定义：前置自增运算符会先将变量的值加 1，然后返回加 1 后的值。
    • 执行顺序：
    • 变量的值先自增。
    • 返回自增后的值。
    • 效率：通常比后置自增效率更高，因为不需要创建临时对象。

    示例

    int x = 5;
int y = ++x; // x 先加 1，然后将加 1 后的值赋给 y
cout << "x = " << x << ", y = " << y << endl;
    输出： 
    x = 6, y = 6

    2. x++（后置自增）

    • 定义：后置自增运算符会先返回变量的当前值，然后再将变量的值加 1。
    • 执行顺序：
    • 返回变量的当前值。
    • 变量的值再自增。
    • 效率：通常比前置自增稍慢，因为需要创建一个临时对象来保存变量的原始值。

    示例

    int x = 5;
int y = x++; // 先将 x 的当前值赋给 y，然后 x 再加 1
cout << "x = " << x << ", y = " << y << endl;
    输出： 
    x = 6, y = 5

    3. 前置和后置自增的实现

    在 C++ 中，自增运算符可以通过运算符重载来实现。以下是它们的实现方式：

    前置自增（++x）

    class Counter {
private:
    int value;

public:
    Counter(int v) : value(v) {}

    // 前置自增
    Counter& operator++() {
        ++value; // 先自增
        return *this; // 返回自增后的当前对象
    }

    void display() const {
        cout << "Value: " << value << endl;
    }
};

    后置自增（x++）

    class Counter {
private:
    int value;

public:
    Counter(int v) : value(v) {}

    // 后置自增
    Counter operator++(int) {
        Counter temp = *this; // 保存当前对象的副本
        ++value; // 自增
        return temp; // 返回原始值（副本）
    }

    void display() const {
        cout << "Value: " << value << endl;
    }
};

    4. 示例对比

    #include <iostream>
using namespace std;

class Counter {
private:
    int value;

public:
    Counter(int v) : value(v) {}

    // 前置自增
    Counter& operator++() {
        ++value;
        return *this;
    }

    // 后置自增
    Counter operator++(int) {
        Counter temp = *this;
        ++value;
        return temp;
    }

    void display() const {
        cout << "Value: " << value << endl;
    }
};

int main() {
    Counter c(5);

    cout << "Using ++c (前置自增):" << endl;
    ++c; // 前置自增
    c.display();

    cout << "Using c++ (后置自增):" << endl;
    c++; // 后置自增
    c.display();

    return 0;
}

    输出： 

    Using ++c (前置自增):
Value: 6
Using c++ (后置自增):
Value: 7

    5. 总结

    特性 前置自增（++x） 后置自增（x++）
    执行顺序 先自增，再返回自增后的值 先返回当前值，再自增
    返回值 自增后的值 自增前的值
    效率 通常更高，因为不需要创建临时对象 通常稍慢，因为需要创建临时对象
    适用场景 更适合需要立即使用自增后的值的场景 更适合需要保留自增前值的场景

    根据具体需求选择使用 ++x 或 x++，在性能敏感的场景下，优先考虑使用前置自增（++x）。

    编译器怎么区分调用前置自增还是后置自增

    1. 前置自增（++x）

    • 调用语法：++x
    • 函数签名： 
      Counter& operator++();
    • 特点：
    • 不带参数。
    • 编译器会匹配到没有参数的 operator++() 函数。

    2. 后置自增（x++）

    • 调用语法：x++
    • 函数签名： 
      Counter operator++(int);
    • 特点：
    • 带一个 int 类型的哑参数。
    • 编译器会匹配到带有 int 参数的 operator++(int) 函数。
    • 这个 int 参数仅用于区分后置自增，调用时不会传递实际的值。

    3. 编译器如何区分？

    调用 ++x（前置自增）

    • 当编译器看到 ++x 时：
    • 它会查找 operator++() 函数（不带参数）。
    • 如果找到匹配的函数，就调用它。

    调用 x++（后置自增）

    • 当编译器看到 x++ 时：
    • 它会查找 operator++(int) 函数（带一个 int 参数）。
    • 如果找到匹配的函数，就调用它。

    内存泄漏和指针悬挂常发生在哪些情况，会有什么危害

    1. 内存泄漏

    定义

    内存泄漏是指程序动态分配的内存未被释放，导致内存资源无法被回收和重用。

    常见情况

    1. 动态分配内存未释放

    2. 使用 new 或 malloc 分配的内存没有使用 delete 或 free 释放。 

      int* ptr = new int(10);
// 忘记 delete ptr，导致内存泄漏

    3. 异常导致未释放

    4. 在动态分配内存后，程序抛出异常，导致释放内存的代码未被执行。 

      int* ptr = new int(10);
throw std::runtime_error("Error"); // 未释放 ptr

    5. 循环引用（智能指针）

    6. 使用 std::shared_ptr 时，两个对象互相引用，导致引用计数无法归零，内存无法释放。 

      struct Node {
    std::shared_ptr<Node> next;
};

    7. 未释放容器中的动态内存

    8. 容器中存储了动态分配的指针，但在容器销毁时未释放这些指针。 
      std::vector<int*> vec;
vec.push_back(new int(10));
// 未释放 vec 中的指针

    危害

    • 内存占用增加：程序运行时间越长，未释放的内存越多，可能导致系统内存耗尽。
    • 性能下降：内存泄漏会导致系统频繁进行内存分配和回收，降低性能。
    • 程序崩溃：当系统内存耗尽时，程序可能无法继续运行。

    2. 指针悬挂

    定义

    指针悬挂是指指针指向的内存已经被释放或超出作用域，但指针仍然保留原来的地址。

    常见情况

    1. 释放内存后未将指针置空

    2. 动态分配的内存被释放后，指针仍然指向原来的地址。 

      int* ptr = new int(10);
delete ptr;
// ptr 仍然指向已释放的内存

    3. 返回局部变量的地址

    4. 函数返回局部变量的地址，局部变量在函数结束后被销毁。 

      int* func() {
    int x = 10;
    return &x; // 返回局部变量地址
}

    5. 多指针指向同一块内存

    6. 多个指针指向同一块内存，其中一个指针释放了内存，其他指针变为悬挂指针。 

      int* ptr1 = new int(10);
int* ptr2 = ptr1;
delete ptr1;
// ptr2 成为悬挂指针

    7. 容器中存储的指针被释放

    8. 容器中存储的指针指向的内存被释放，但容器中的指针仍然存在。 
      std::vector<int*> vec;
int* ptr = new int(10);
vec.push_back(ptr);
delete ptr;
// vec 中的指针悬挂

    危害

    • 未定义行为：访问悬挂指针可能导致程序崩溃或产生不可预测的行为。
    • 数据损坏：悬挂指针可能指向已被其他对象使用的内存，导致数据被意外修改。
    • 调试困难：悬挂指针的问题通常难以定位，可能在程序运行的某个随机时刻触发。

    如何避免内存泄漏和指针悬挂

    1. 避免内存泄漏

    • 使用智能指针：
    • 使用 std::unique_ptr 或 std::shared_ptr 管理动态内存，避免手动释放。 
      std::unique_ptr<int> ptr = std::make_unique<int>(10);
    • 及时释放内存：
    • 在动态分配内存后，确保在适当的时机释放内存。 
      int* ptr = new int(10);
delete ptr;
    • 使用 RAII（资源获取即初始化）：
    • 将资源的分配和释放封装在类中，利用析构函数自动释放资源。 
      class Resource {
    int* ptr;
public:
    Resource() : ptr(new int(10)) {}
    ~Resource() { delete ptr; }
};

    2. 避免指针悬挂

    • 释放后置空：
    • 在释放动态内存后，将指针置为 nullptr。 
      int* ptr = new int(10);
delete ptr;
ptr = nullptr;
    • 避免返回局部变量的地址：
    • 如果需要返回数据，使用动态分配或传递引用。 
      int* func() {
    return new int(10); // 返回动态分配的内存
}
    • 使用智能指针：
    • 智能指针会自动管理内存，避免悬挂指针问题。 
      std::shared_ptr<int> ptr1 = std::make_shared<int>(10);
std::shared_ptr<int> ptr2 = ptr1; // 自动管理引用计数

    总结

    问题 常见情况 危害
    内存泄漏 动态分配未释放、异常中断、循环引用、容器中动态内存未释放 内存占用增加、性能下降、程序崩溃
    指针悬挂 释放后未置空、返回局部变量地址、多指针指向同一内存、容器中指针被释放 未定义行为、数据损坏、调试困难

    通过使用智能指针、RAII 和良好的编码习惯，可以有效避免内存泄漏和指针悬挂问题，提高程序的稳定性和可维护性。
