    5月16日基础知识

    什么是C++ STL？

    C++ STL（Standard Template Library，标准模板库）是C++标准库中的一个重要组成部分。它提供了一组通用的类和函数模板，用于实现常用的数据结构和算法。STL主要包括以下几个部分：

    1. 容器（Containers）：如vector、list、deque、set、map等，用于存储和管理数据。
    2. 算法（Algorithms）：如排序、查找、拷贝等操作，可以作用于各种容器。
    3. 迭代器（Iterators）：用于遍历容器中的元素，类似于指针。
    4. 函数对象（Function Objects）：可以像函数一样使用的对象，常用于自定义排序等。
    5. 适配器（Adapters）：如stack、queue、priority_queue等，对已有容器进行功能扩展。

    什么时候需要用hash_map，什么时候需要用map?

    hash_map（C++11标准中为unordered_map）和map都是STL中的关联容器，但它们的底层实现和适用场景不同：

    • map：底层通常用红黑树实现，键值是有序的，支持有序遍历。查找、插入、删除的时间复杂度为O(log n)。适用于需要按顺序遍历元素或需要区间操作的场景。

    • hash_map/unordered_map：底层用哈希表实现，键值是无序的，查找、插入、删除的平均时间复杂度为O(1)。适用于只关心查找效率、不需要有序遍历的场景。

    总结： - 需要有序数据或区间操作时，用map。 - 只关心查找效率且不需要有序时，用unordered_map（即hash_map）。

    STL中hashtable的底层实现？

    STL中hashtable的底层实现主要基于哈希表（Hash Table），其核心思想如下：

    1. 数组+链表（拉链法）：
      哈希表内部维护一个数组，每个数组元素称为“桶”。当多个元素哈希到同一个桶时，这些元素会以链表（或其他结构，如C++11中的链表或红黑树）形式存储在该桶中。

    2. 哈希函数：
      通过哈希函数将键映射到数组的下标，决定元素存储的位置。哈希函数的好坏直接影响查找效率。

    3. 冲突解决：
      常用拉链法（链表法）解决哈希冲突，即同一个桶中的元素用链表连接。

    4. 动态扩容：
      当元素数量增多导致哈希表负载因子过高时，会自动扩容（增加桶的数量），并重新分配已有元素（rehash）。

    5. 时间复杂度：
      查找、插入、删除的平均时间复杂度为O(1)，最坏情况下为O(n)。

    示意图：

    哈希表（数组）
+-----+-----+-----+-----+-----+
|  0  |  1  |  2  |  3  | ... |
+--|--+--|--+--|--+--|--+-----+
   |     |     |     |
  链表  链表  链表  链表

    在C++11及以后，unordered_map和unordered_set就是基于hashtable实现的。

    vector 底层原理及其相关面试题

    vector 底层原理

    vector 是C++ STL中最常用的顺序容器之一，其底层实现基于动态数组。主要原理如下：

    1. 动态数组：
      vector 内部维护一个连续的内存空间（类似C语言的数组），可以通过下标随机访问元素，访问效率为O(1)。

    2. 自动扩容：
      当插入新元素且空间不足时，vector 会自动分配更大的内存（通常为原容量的2倍），并将原有元素拷贝到新空间，释放旧空间。扩容时的插入操作复杂度为O(n)，但均摊下来每次插入的平均复杂度为O(1)。

    3. 内存管理：
      vector 的容量（capacity）和实际元素个数（size）是分开的。容量大于等于元素个数，只有当size等于capacity时才会扩容。

    4. 元素存储：
      元素在内存中是连续存储的，保证了高效的随机访问和良好的缓存局部性。

    5. 插入和删除：

    6. 尾部插入/删除（push_back/pop_back）：效率高，O(1)均摊。
    7. 中间或头部插入/删除：需要移动大量元素，效率低，O(n)。

    常见面试题

    1. vector和数组的区别？
    2. vector能动态扩容，数组大小固定。
    3. vector有丰富的成员函数，数组没有。

    4. vector可以自动管理内存，数组需要手动管理。

    5. vector扩容机制？

    6. 一般按2倍扩容（实现相关），扩容时会分配新空间并拷贝旧数据，释放旧空间。

    7. vector的插入和删除效率？

    8. 尾部插入/删除效率高，O(1)均摊。

    9. 头部或中间插入/删除效率低，O(n)。

    10. vector为什么不能高效地在头部插入元素？

    11. 因为所有元素都要整体向后移动，时间复杂度为O(n)。

    12. 如何释放vector多余的内存？

    13. 可以用shrink_to_fit()或swap技巧：vector<T>(v).swap(v);

    14. vector的迭代器失效场景？

    15. 扩容、插入或删除元素时，原有的迭代器可能会失效。

    示例代码：vector自动扩容

    #include <vector>
#include <iostream>
using namespace std;

int main() {
    vector<int> v;
    for (int i = 0; i < 10; ++i) {
        v.push_back(i);
        cout << "size: " << v.size() << ", capacity: " << v.capacity() << endl;
    }
    return 0;
}

    list 底层原理及其相关面试题

    list 是C++ STL中的双向链表容器，其底层实现基于双向链表。主要原理如下：

    1. 双向链表结构：
      每个节点包含数据和两个指针，分别指向前驱和后继节点。这样可以在常数时间内在任意位置插入和删除元素。

    2. 非连续内存：
      list 的元素在内存中不是连续存储的，每个节点单独分配空间，因此不支持随机访问（不能用下标访问）。

    3. 插入和删除效率：
      在已知位置（通过迭代器）插入和删除元素的时间复杂度为O(1)，不需要移动其他元素。

    4. 遍历效率：
      由于不是连续内存，遍历效率比vector低，且不支持高效的随机访问。

    5. 空间开销：
      每个节点除了存储数据外，还需要额外存储两个指针，空间开销较大。

    常见面试题

    1. list和vector的区别？
    2. list底层是双向链表，vector是动态数组。
    3. list插入/删除效率高（O(1)），但不支持随机访问。

    4. vector支持高效随机访问，尾部插入/删除快，但中间操作慢。

    5. list的插入和删除为什么快？

    6. 只需修改相邻节点的指针，不需要移动其他元素，时间复杂度O(1)。

    7. list的遍历为什么慢？

    8. 节点分散在内存中，缓存命中率低，且只能顺序访问。

    9. list的迭代器失效场景？

    10. 插入不会导致其他迭代器失效，删除某节点后指向该节点的迭代器失效。

    11. list适合哪些场景？

    12. 需要频繁在中间插入/删除元素且不关心随机访问的场景。

    示例代码：list插入和删除

    #include <list>
#include <iostream>
using namespace std;

int main() {
    list<int> l = {1, 2, 3};
    auto it = l.begin();
    ++it;
    l.insert(it, 10); // 在第二个位置插入10
    l.erase(it);      // 删除原来的第二个元素
    for (int x : l) cout << x << " ";
    return 0;
}

    deque底层原理及其相关面试题

    deque（双端队列）是C++ STL中的一种顺序容器，支持在头尾两端高效插入和删除。其底层实现原理如下：

    1. 分段连续内存结构（块表结构）：
      deque不是像vector那样用一块连续的大内存，而是由多个小的连续内存块（称为“缓冲区”或“块”）组成，并用一个指针数组（map）管理这些块。这样既能支持高效的头尾操作，又避免了频繁的整体内存搬移。

    2. 高效的头尾操作：
      在头部或尾部插入/删除元素时，只需在块的两端操作或分配/释放新的块，时间复杂度为O(1)均摊。

    3. 随机访问：
      deque支持下标随机访问，时间复杂度为O(1)。实现时通过块表结构和指针偏移计算定位到具体元素。

    4. 内存管理：
      当头尾空间不足时，会自动分配新的块，并调整指针数组（map）。

    5. 插入/删除效率：

    6. 头尾插入/删除：O(1)均摊。
    7. 中间插入/删除：需要移动元素，效率为O(n)。

    常见面试题

    1. deque和vector的区别？
    2. deque支持头尾高效插入/删除，vector只支持尾部高效操作。
    3. deque底层是分段连续内存，vector是单块连续内存。

    4. deque随机访问效率和vector一样，都是O(1)。

    5. deque为什么能高效地在头部插入元素？

    6. 因为其内存结构允许在头部分配新块，无需整体移动元素。

    7. deque的空间管理方式？

    8. 采用指针数组（map）管理多个小块，每个块存储一部分元素。

    9. deque适合哪些场景？

    10. 需要频繁在头尾两端插入/删除元素的场景，如实现队列、滑动窗口等。

    11. deque的迭代器失效场景？

    12. 在头尾插入/删除时，只有涉及到的块的迭代器失效；整体扩容或收缩时，所有迭代器可能失效。

    vector如何释放空间

    可以通过以下两种方式释放vector多余的空间：

    1. shrink_to_fit() 方法
      调用 v.shrink_to_fit();，请求将vector的容量缩小到与当前元素个数一致（但不是强制，是否真的缩容由实现决定）。

    2. swap 技巧
      使用临时对象和swap强制释放多余空间： 

      vector<int>(v).swap(v);
      这样会创建一个只包含当前元素的新vector，并与原vector交换，原来的多余空间会被释放。

    这两种方法都可以有效减少vector的内存占用。

    如何在共享内存上使用STL标准库？

    在共享内存上直接使用STL标准库容器（如vector、map等）通常不可行，因为STL容器内部会动态分配内存（如new），这些内存只在当前进程有效，其他进程无法访问。

    正确做法

    1. 使用专门支持共享内存的STL容器实现
      推荐使用 Boost.Interprocess 库，它提供了可以在共享内存上使用的STL兼容容器，如boost::interprocess::vector、map等。

    2. 基本步骤示例

    #include <boost/interprocess/managed_shared_memory.hpp>
#include <boost/interprocess/containers/vector.hpp>
#include <boost/interprocess/allocators/allocator.hpp>

using namespace boost::interprocess;

int main() {
    // 创建或打开共享内存
    managed_shared_memory shm(create_only, "MySharedMemory", 1024 * 1024);

    // 定义分配器
    typedef allocator<int, managed_shared_memory::segment_manager> ShmAllocator;
    typedef vector<int, ShmAllocator> ShmVector;

    // 在共享内存上构造vector
    ShmVector *vec = shm.construct<ShmVector>("MyVector")(shm.get_segment_manager());

    // 使用vector
    vec->push_back(1);
    vec->push_back(2);

    return 0;
}
    1. 注意事项
    2. 只能用支持共享内存分配器的容器。
    3. 指针、引用等成员不能指向进程私有内存。
    4. 需要手动管理共享内存的创建和销毁。

    总结：
    要在共享内存上用STL容器，推荐用Boost.Interprocess库的容器和分配器，不能直接用标准STL容器。

    map插入方式有哪几种？

    map插入元素常用的方式有以下几种：

    1. insert(pair) 

      m.insert(std::make_pair(key, value));
m.insert(std::pair<Key, Value>(key, value));

    2. insert(value_type) 

      m.insert(map<Key, Value>::value_type(key, value));

    3. 下标运算符[] 

      m[key] = value;
      如果key不存在，会自动插入新元素。

    4. emplace 

      m.emplace(key, value);
      直接在容器内部原地构造，提高效率。

    5. insert带hint版本 

      auto it = m.begin();
m.insert(it, std::make_pair(key, value));

    总结：
    常用的有insert、emplace、[]三种，实际开发中推荐用emplace或[]。

    下面是一个实际的 C++ 程序例子，演示了map的多种插入方式：

    #include <iostream>
#include <map>
using namespace std;

int main() {
    map<int, string> m;

    // 1. insert(pair)
    m.insert(make_pair(1, "apple"));
    m.insert(pair<int, string>(2, "banana"));

    // 2. insert(value_type)
    m.insert(map<int, string>::value_type(3, "cherry"));

    // 3. 下标运算符[]
    m[4] = "date";

    // 4. emplace
    m.emplace(5, "elderberry");

    // 5. insert带hint
    auto it = m.begin();
    m.insert(it, make_pair(6, "fig"));

    // 输出所有元素
    for (const auto& p : m) {
        cout << p.first << ": " << p.second << endl;
    }

    return 0;
}

    输出示例： 

    1: apple
2: banana
3: cherry
4: date
5: elderberry
6: fig

    这个例子展示了map的所有常见插入方式。
