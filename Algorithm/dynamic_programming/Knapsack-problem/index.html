    背包问题

    完全背包问题是动态规划中的经典问题之一。它与 0-1 背包问题类似，但有一个关键的区别：在完全背包问题中，每种物品可以被选择多次，而不是最多只能选择一次。

    0-1背包(NPC问题)

    在 0-1 背包问题 中，每种物品只能选择一次，状态转移方程如下：

    状态定义

    • dp[i][j] 表示前 i 个物品在背包容量为 j 时的最大价值。

    状态转移方程

    对于每种物品 i： - 如果不选择物品 i，则： 

    dp[i][j] = dp[i - 1][j];
    表示当前容量 j 的最大价值等于前 i-1 个物品在容量 j 时的最大价值。

    • 如果选择物品 i，则： 

      dp[i][j] = dp[i - 1][j - weight[i]] + value[i];
      表示当前容量 j 的最大价值等于前 i-1 个物品在容量 j - weight[i] 时的最大价值，加上物品 i 的价值。

    • 综合两种情况，取最大值： 

      dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - weight[i]] + value[i]);

    初始化

    • dp[0][j] = 0：表示没有物品时，背包的最大价值为 0。
    • dp[i][0] = 0：表示背包容量为 0 时，最大价值为 0。

    优化为一维数组

    如果使用一维数组优化，状态转移方程为： 

    dp[j] = max(dp[j], dp[j - weight[i]] + value[i]);
    注意：在一维数组中，内层循环需要从 大到小 遍历 j，以避免重复使用同一个物品。

    完全背包

    给定一个容量为 W 的背包和 n 种物品，每种物品有以下属性： - 重量 weight[i] - 价值 value[i]

    目标是从这些物品中选择若干个，使得在不超过背包容量 W 的前提下，背包中的总价值最大化。

    与 0-1 背包问题不同的是，每种物品可以被选择任意次（即可以重复放入背包）。

    动态规划解法

    状态定义

    设 dp[j] 表示背包容量为 j 时的最大价值。

    状态转移方程

    对于每种物品 i： - 如果选择该物品，则状态转移为 dp[j] = max(dp[j], dp[j - weight[i]] + value[i])。

    这里的 dp[j - weight[i]] + value[i] 表示当前容量 j 减去物品 i 的重量后，剩余容量的最大价值加上物品 i 的价值。

    初始化

    • dp[0] = 0，表示背包容量为 0 时的最大价值为 0。
    • 其他 dp[j] 初始化为 0 或负无穷（根据题目是否允许部分容量未被使用）。

    二维数组加三重循环

    1. 二维数组定义：

    2. dp[i][j] 表示前 i 个物品在背包容量为 j 时的最大价值。

    3. 三重循环：

    4. 第一层循环：遍历物品 i。
    5. 第二层循环：遍历背包容量 j。

    6. 第三层循环（隐含在状态转移中）：遍历物品选择数量j/w[i]考虑当前物品可以被选择多次。

    7. 状态转移方程：

    8. 不选当前物品：dp[i][j] = dp[i - 1][j] 在最内层循环之前一定要初始化dp[i][j]为dp[i - 1][j]

    9. 选当前物品：dp[i][j] = max(dp[i][j], dp[i][j - k*weight[i]] + k*value[i])记住每次遍历物品数量k时是跟上个数量k-1的dp[i][j]对比。

    10. 初始化：

    11. dp[0][j] = 0 表示没有物品时，背包的最大价值为 0。
    12. dp[i][0] = 0 表示背包容量为 0 时，最大价值为 0。

    硬币找零问题

    设有n种不同面值的硬币，各硬币的面值存于数组T[1 : n] 中。现要用这些面值的硬币来 找钱。可以使用的各种面值的硬币个数存于数组Coins[1 : n]中。对任意钱数0 ≤ m ≤ 20001，设计一个用最少硬币找钱m 的方法。

    使用动态规划解决硬币找零问题，目标是用最少的硬币凑成指定金额 m，同时考虑每种硬币的使用数量限制。以下是你的算法的复述：

    1. 问题建模

    • 定义一个二维数组 dp[i][j]：
    • 表示用前 i + 1 种硬币凑成金额 j 所需的最少硬币数。
    • 定义一个二维数组 choice[i][j]：
    • 记录凑成金额 j 时，第 i 种硬币的使用数量。

    2. 初始化

    • 初始化 dp[0][j]：
    • 如果金额 j 是第 0 种硬币面值的倍数，且使用的硬币数量不超过限制，则可以直接计算出所需硬币数。
    • 否则，设置为无穷大（表示无法凑成金额 j）。
    • 初始化 choice[0][j]：
    • 记录第 0 种硬币的使用数量。

    3. 状态转移

    • 从第 1 种硬币开始，逐步更新 dp[i][j]：
    • 对于每种硬币 i，尝试使用 0 到最大数量 coins[i] 的硬币。
    • 更新 dp[i][j] 为当前最优解（即最少硬币数）。
    • 同时记录当前硬币的使用数量到 choice[i][j]。

    状态转移方程：

    dp[i][j] = min(dp[i][j], dp[i − 1][j − k × T[i]] + k)
    - 其中： - 0 ≤ k ≤ coins[i]：表示当前硬币的使用数量。 - j − k × T[i] ≥ 0：表示当前金额 j 可以通过使用 k 枚第 i 种硬币凑成。

    记录选择：

    choice[i][j] = k
    - 表示凑成金额 j 时，第 i 种硬币使用了 k 枚。

    4. 回溯结果

    • 最终通过 dp[n−1][m] 判断是否可以凑成金额 m：
    • 如果 dp[n−1][m] 是有限值，则表示可以凑成金额 m。

    • 如果 dp[n−1][m] 是无穷大，则表示无法凑成金额 m。

    • 如果可以凑成金额 m，通过回溯 choice 数组，逐步还原每种硬币的使用数量，并输出结果。

    5. 算法总结

    • 动态规划表 dp：
    • 用于记录凑成每个金额所需的最少硬币数。
    • 选择表 choice：
    • 用于记录每种硬币的使用数量，方便回溯结果。
    • 状态转移：
    • 尝试使用当前硬币的不同数量，更新最优解。
    • 回溯：
    • 从 choice 数组中还原每种硬币的使用数量。
    // 初始化 dp 和 choice 数组
    vector<vector<int>> dp(n, vector<int>(m + 1, INT_MAX));
    vector<vector<int>> choice(n, vector<int>(m + 1, 0));

    dp[0][0] = 0; // 凑成金额 0 不需要硬币
    for (int i = 0; i <= m; i++) {
        for (int k = 0; k <= coins[0]; k++) {
            if (k * T[0] == i) {
                dp[0][i] = k;
                choice[0][i] = k; // 第 0 种硬币凑成金额 i 的数量
            }
        }
    }

    // 动态规划
    for (int i = 1; i < n; i++) {
        for (int j = 0; j <= m; j++) {
            dp[i][j] = dp[i-1][j]; // 不使用第 i 种硬币
            choice[i][j] = 0;      // 默认不使用第 i 种硬币
            for (int k = 0; k <= coins[i]; k++) {
                if (j >= k * T[i] && dp[i-1][j - k * T[i]] != INT_MAX) {
                    if (dp[i][j] > dp[i-1][j - k * T[i]] + k) {
                        dp[i][j] = dp[i-1][j - k * T[i]] + k;
                        choice[i][j] = k; // 记录第 i 种硬币的使用数量
                    }
                }
            }
        }
    }
