    子序列问题

    最长上升子序列（LIS）

    问题描述

    给定一个长度为 n 的数组,找到一个最长的子序列,使得这个子序列是严格递增的。

    动态规划解法

    1. 状态定义：

    2. dp[i] 表示以第 i 个元素结尾的最长上升子序列的长度。

    3. 状态转移方程：

    4. 对于每个 j < i,如果 nums[j] < nums[i],则： 
      dp[i] = max(dp[i], dp[j] + 1);

    5. 表示当前元素可以接在 j 的子序列后面。

    6. 初始化：

    7. 每个元素单独作为子序列时,长度为 1： 

      dp[i] = 1;

    8. 结果：

    9. 最终结果是 dp 数组中的最大值,即： 
      LIS = max(dp[i]) for all i.

    时间复杂度

    • 动态规划解法：O(n^2)。
    • 使用二分查找优化：O(n log n)。

    最长公共子序列（LCS）

    问题描述

    给定两个字符串 text1 和 text2,找到它们的最长公共子序列的长度。子序列是指可以从一个字符串中删除一些字符（或不删除）后得到的另一个字符串。

    动态规划解法

    1. 状态定义：

    2. dp[i][j] 表示 text1[0...i-1] 和 text2[0...j-1] 的最长公共子序列长度。

    3. 状态转移方程：

    4. 如果 text1[i-1] == text2[j-1],则： 
      dp[i][j] = dp[i-1][j-1] + 1;

    5. 否则： 

      dp[i][j] = max(dp[i-1][j], dp[i][j-1]);

    6. 初始化：

    7. 如果 i == 0 或 j == 0,则： 

      dp[i][j] = 0;

    8. 结果：

    9. 最终结果是 dp[m][n],其中 m 和 n 分别是 text1 和 text2 的长度。

    时间复杂度

    • 时间复杂度：O(m * n)。
    • 空间复杂度：O(m * n),可以优化为 O(min(m, n))。

    最长回文子串

    动态规划的核心思想

    1. 定义状态：

    2. 用 dp[i][j] 表示字符串从索引 i 到 j 是否是回文子串。

      • dp[i][j] = true 表示 s[i..=j] 是回文子串。
      • dp[i][j] = false 表示 s[i..=j] 不是回文子串。

    3. 状态转移方程：

    4. 如果 s[i] == s[j],则：
      • 当子串长度为 1 或 2 时（即 j - i <= 1）,dp[i][j] = true。
      • 否则,dp[i][j] = dp[i+1][j-1]。

    5. 如果 s[i] != s[j],则 dp[i][j] = false。

    6. 初始化：

    7. 单个字符一定是回文子串,因此 dp[i][i] = true。

    8. 结果：

    9. 遍历所有可能的子串,找到最长的回文子串。 从长度len为2开始遍历到长度为n,用变量start和maxlen记录最长回文子串的开始位置和最长长度,开始len=2时,相当于在字符串s中按个遍历相邻两对字符s[i]和s[j],判断两个是否相等相等dp[i][j]为true,然后len=3后,判断s[i]是否等于s[j],若相等,则判断dp[i+1][j-1]是否为true,也就是判断i,j之间的这些字符串是否为回文字符串。 核心代码: 
              for(int len=2;len<=n;len++){
            for(int i=0;i<=n-len;i++){
                int j=i+len-1;
                if(s[i]==s[j]&&(len==2 || dp[i+1][j-1])){
                    dp[i][j]=true;
                    if(len>maxlen){
                        start=i;
                        maxlen=len;
                    }
                }
            }
        }

    只能交易两次,我本来打算计算序列中所有连续的升序的区间的利润,然后把这些利润存入一个向量,然后排序,找出最大的两项。但是编译错误,但可以在本地运行一些测试用例通过,不知道什么原因,暂且不管,感觉这个有点像贪心思想。

    正确做法应该是动态规划,用两个动态规划数组left 和right数组,分别表示第0-i天的利润和第i-n的利润。

    你的代码在执行时可能会出错,主要原因是 数组越界 和 逻辑问题。以下是详细分析和修复建议：

    问题 1：数组越界

    在 while 循环中： 

    while (prices[i] - prices[i - 1] > 0) {
    temp += prices[i] - prices[i - 1];
    i++;
}
    - 当 i 增加到 prices.size() 时,prices[i] 会访问越界的内存,导致运行时错误。

    修复方法：

    在进入 while 循环之前,检查 i 是否越界： 

    while (i < prices.size() && prices[i] - prices[i - 1] > 0) {
    temp += prices[i] - prices[i - 1];
    i++;
}

    问题 2：逻辑问题

    我原来想通过计算所有的上升区间利润,并取最大的两个利润相加。 但是如果是1,3,2,7,2,0,9这个序列按照我的算法就会出错。

    动态规划思路：

    1. 定义状态：

    2. 用两个数组 left 和 right：

      • left[i] 表示从第 0 天到第 i 天的最大利润。
      • right[i] 表示从第 i 天到最后一天的最大利润。

    3. 状态转移：

    4. 计算 left[i]：
      • 遍历 prices,记录最低价格 minPrice,并更新最大利润： 
        left[i] = max(left[i - 1], prices[i] - minPrice);

    5. 计算 right[i]：

      • 从右向左遍历 prices,记录最高价格 maxPrice,并更新最大利润： 
        right[i] = max(right[i + 1], maxPrice - prices[i]);

    6. 合并结果：

    7. 遍历所有可能的分割点 i,计算最大利润： 

      maxProfit = max(maxProfit, left[i] + right[i]);

    8. 左侧最大利润：

    9. 遍历 prices,记录最低价格 minPrice,并更新从第 0 天到第 i 天的最大利润。

    10. 右侧最大利润：

    11. 从右向左遍历 prices,记录最高价格 maxPrice,并更新从第 i 天到最后一天的最大利润。

    12. 合并结果：

    13. 遍历所有可能的分割点 i,将 left[i] 和 right[i] 相加,得到最多两次交易的最大利润。
            // 左侧最大利润
        vector<int> left(n, 0);
        int minPrice = prices[0];
        for (int i = 1; i < n; i++) {
            minPrice = min(minPrice, prices[i]);
            left[i] = max(left[i - 1], prices[i] - minPrice);
        }

        // 右侧最大利润
        vector<int> right(n, 0);
        int maxPrice = prices[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            maxPrice = max(maxPrice, prices[i]);
            right[i] = max(right[i + 1], maxPrice - prices[i]);
        }

        // 合并结果
        int maxProfit = 0;
        for (int i = 0; i < n; i++) {
            maxProfit = max(maxProfit, left[i] + right[i]);
        }

    lc72编辑距离

    给你两个单词 word1 和 word2， 请返回将 word1 转换成 word2 所使用的最少操作数 。

    你可以对一个单词进行如下三种操作：

    插入一个字符 删除一个字符 替换一个字符

    这个题需要动态规划求解，但是怎么建立状态转移方程呢 定义 dp[i][j] dp[i][j] 代表 word1 中前 i 个字符，变换到 word2 中前 j个字符，最短需要操作的次数

    需要考虑 word1或 word2 一个字母都没有，即全增加/删除的情况，所以预留 dp[0][j] 和 dp[i][0] 状态转移 增，dp[i][j] = dp[i][j - 1] + 1 删，dp[i][j] = dp[i - 1][j] + 1 改，dp[i][j] = dp[i - 1][j - 1] + 1 按顺序计算，当计算 dp[i][j] 时，dp[i - 1][j] ， dp[i][j - 1] ， dp[i - 1][j - 1] 均已经确定了 配合增删改这三种操作，需要对应的 dp 把操作次数加一，取三种的最小 如果刚好这两个字母相同 word1[i - 1] = word2[j - 1] ，那么可以直接参考 dp[i - 1][j - 1] ，操作不用加一

    class Solution {
public:
    int minDistance(string word1, string word2) {
        int m = word1.size();
        int n = word2.size();
        // dp[i][j] 表示将 word1 的前 i 个字符转换为 word2 的前 j 个字符所需的最少操作数
        vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));

        // 初始化 dp 数组
        for (int i = 0; i <= m; i++) dp[i][0] = i; // 删除操作
        for (int j = 0; j <= n; j++) dp[0][j] = j; // 插入操作

        // 填充 dp 数组
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1[i - 1] == word2[j - 1]) {
                    // 如果当前字符相等，不需要额外操作
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    // 否则，取增、删、改三种操作的最小值
                    dp[i][j] = min(dp[i - 1][j - 1] + 1, // 替换操作
                                   min(dp[i][j - 1] + 1,  // 插入操作
                                       dp[i - 1][j] + 1)); // 删除操作
                }
            }
        }

        return dp[m][n];
    }
};
