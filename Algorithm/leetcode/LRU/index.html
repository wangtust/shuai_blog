    LRU缓存机制

    LRU（Least Recently Used）缓存机制是一种缓存淘汰策略，用于在缓存空间有限的情况下，决定哪些数据需要被移除，以便为新数据腾出空间。

    基本思想

    LRU 的核心思想是：优先淘汰最近最少使用的数据。也就是说，如果某个数据最近被访问过，那么它在未来被访问的可能性更高，而那些很久没有被访问的数据会被优先移除。

    工作原理

    1. 缓存会维护一个有序的数据结构（通常是链表或哈希表+双向链表）。
    2. 每次访问数据时：
    3. 如果数据已经在缓存中，则将其标记为最近使用（通常移动到链表头部）。
    4. 如果数据不在缓存中：
      • 如果缓存未满，则将数据插入缓存。
      • 如果缓存已满，则移除最近最少使用的数据（链表尾部），然后插入新数据。
    5. 通过这种方式，缓存始终保持最近使用的数据在前，最少使用的数据在后。

    实现方式

    LRU 通常使用以下两种数据结构实现： 1. 双向链表：用于维护数据的访问顺序。 2. 哈希表：用于快速定位数据是否在缓存中。

    伪代码

    以下是 LRU 缓存的基本操作： - get(key)：获取缓存中的数据。 - put(key, value)：插入或更新缓存中的数据。

    class LRUCache {
private:
    int capacity;
    list<pair<int, int>> cache; // 双向链表存储键值对
    unordered_map<int, list<pair<int, int>>::iterator> map; // 哈希表存储键及其在链表中的位置

public:
    LRUCache(int capacity) : capacity(capacity) {}

    int get(int key) {
        if (map.find(key) == map.end()) return -1; // 不存在
        // 将访问的节点移到链表头部
        cache.splice(cache.begin(), cache, map[key]);
        return map[key]->second;
    }

    void put(int key, int value) {
        if (map.find(key) != map.end()) {
            // 如果键已存在，更新值并移到链表头部
            map[key]->second = value;
            cache.splice(cache.begin(), cache, map[key]);
        } else {
            // 如果键不存在，检查容量
            if (cache.size() == capacity) {
                // 移除链表尾部节点
                map.erase(cache.back().first);
                cache.pop_back();
            }
            // 插入新节点到链表头部
            cache.emplace_front(key, value);
            map[key] = cache.begin();
        }
    }
};

    应用场景

    1. 操作系统的内存管理：如页面置换算法。
    2. 数据库缓存：如 Redis 的缓存淘汰策略。
    3. Web 缓存：浏览器缓存中经常使用 LRU 策略。

    LRU 是一种简单且高效的缓存淘汰策略，适用于大多数需要缓存的场景。

    哈希表（Hash Table）是一种基于哈希函数的数据结构，用于以常数时间复杂度 ( O(1) ) 实现快速的数据存储和查找。它通过将键值对映射到一个数组的索引位置来存储数据。

    基本原理

    1. 哈希函数：
      哈希表使用一个哈希函数将键（Key）映射到数组中的索引位置。
      [ index = hash(key) ] 哈希函数的目标是将不同的键尽可能均匀地分布到数组中。

    2. 存储结构：
      哈希表通常由一个数组和一个哈希函数组成，数组的每个位置称为一个“桶”（Bucket）。

    3. 冲突处理：
      当两个键通过哈希函数映射到相同的索引位置时，就会发生冲突。常见的冲突解决方法有：

    4. 链地址法：每个桶存储一个链表，冲突的元素存储在链表中。
    5. 开放地址法：当发生冲突时，寻找下一个空闲位置存储数据。

    操作

    1. 插入（Insert）：
      使用哈希函数计算键的索引，将数据存储到对应的桶中。
    2. 查找（Search）：
      使用哈希函数计算键的索引，直接访问对应的桶。
    3. 删除（Delete）：
      使用哈希函数定位数据所在的桶，然后删除。

    优点

    • 查找、插入和删除的时间复杂度接近 ( O(1) )。
    • 适合快速查找和存储大量数据。

    缺点

    • 哈希函数设计不当可能导致冲突过多，影响性能。
    • 需要额外的空间来存储桶和链表。
    • 不支持有序数据存储。

    应用场景

    1. 缓存：如 LRU 缓存机制中的快速查找。
    2. 数据库索引：用于快速定位数据。
    3. 集合和映射：如 C++ 中的 std::unordered_map 和 std::unordered_set。

    哈希表是现代计算机科学中非常重要的数据结构，广泛应用于需要快速查找的场景。

    以下是一个简单的 C++ 哈希表实现，使用链地址法（链表）来处理哈希冲突：

    #include <iostream>
#include <list>
#include <vector>
#include <string>

class HashTable {
private:
    // 哈希表的大小
    int capacity;
    // 使用向量存储链表，每个链表是一个桶
    std::vector<std::list<std::pair<std::string, int>>> table;

    // 哈希函数：将字符串键映射到索引
    int hashFunction(const std::string& key) {
        int hash = 0;
        for (char ch : key) {
            hash = (hash * 31 + ch) % capacity; // 简单的哈希算法
        }
        return hash;
    }

public:
    // 构造函数，初始化哈希表
    HashTable(int size) : capacity(size), table(size) {}

    // 插入键值对
    void insert(const std::string& key, int value) {
        int index = hashFunction(key);
        for (auto& pair : table[index]) {
            if (pair.first == key) {
                pair.second = value; // 如果键已存在，更新值
                return;
            }
        }
        table[index].emplace_back(key, value); // 插入新键值对
    }

    // 查找键对应的值
    int search(const std::string& key) {
        int index = hashFunction(key);
        for (const auto& pair : table[index]) {
            if (pair.first == key) {
                return pair.second; // 找到键，返回值
            }
        }
        return -1; // 未找到，返回 -1
    }

    // 删除键值对
    void remove(const std::string& key) {
        int index = hashFunction(key);
        for (auto it = table[index].begin(); it != table[index].end(); ++it) {
            if (it->first == key) {
                table[index].erase(it); // 删除键值对
                return;
            }
        }
    }

    // 打印哈希表内容（用于调试）
    void print() {
        for (int i = 0; i < capacity; ++i) {
            std::cout << "Bucket " << i << ": ";
            for (const auto& pair : table[i]) {
                std::cout << "[" << pair.first << ": " << pair.second << "] ";
            }
            std::cout << "\n";
        }
    }
};

int main() {
    HashTable hashTable(10); // 创建一个大小为 10 的哈希表

    // 插入键值对
    hashTable.insert("Alice", 25);
    hashTable.insert("Bob", 30);
    hashTable.insert("Charlie", 35);

    // 打印哈希表
    hashTable.print();

    // 查找键
    std::cout << "Value for 'Alice': " << hashTable.search("Alice") << "\n";
    std::cout << "Value for 'Bob': " << hashTable.search("Bob") << "\n";

    // 删除键
    hashTable.remove("Alice");
    hashTable.print();

    return 0;
}

    代码说明

    1. 哈希函数：
    2. 使用简单的字符串哈希算法，将字符串键映射到哈希表的索引。

    3. 通过 % capacity 确保索引在表的范围内。

    4. 链地址法：

    5. 每个桶是一个链表，用于存储冲突的键值对。

    6. 插入、查找和删除操作都在链表中进行。

    7. 主要操作：

    8. insert：插入键值对，如果键已存在则更新值。
    9. search：查找键对应的值，未找到返回 -1。

    10. remove：删除指定键的键值对。

    11. 调试功能：

    12. print 方法打印哈希表的内容，便于观察哈希表的状态。
